So you need to know the motivation behind this code base. This code base was created in order to improve the performance of co-folding models such as Boltz 2. As many papers have pointed out, these models have memorized issues. They don't really learn the physics well. The question then becomes how can you make the models learn physics better? Now, it's a data driven model in the end, right? So the idea is to feed it to reference data that would be consistent to the generative structure of this model. In the end, these models are diffusion models. They give in some input generate some structure. Physically consistent results have to obey a Boltzmann distribution with the given energy function. So my idea is to do supervised fine tuning using enhanced sampling data. The second part of my hypothesis is the fact is the fact is you need evaluations that really probe the failure modes of the given model. How are its tails behaving? Does it predict metastable basins over the CV space? That's why I came up with an algorithm to push general models to perform enhanced sampling.
